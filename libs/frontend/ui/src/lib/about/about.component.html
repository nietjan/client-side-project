Sportschool Projectcasus

Deze webapplicatie is gemaakt in het kader van het vak Client-side web developtment, individueel project.

Beschrijving:

Er is een sportschool met meerdere locaties. Elke locatie is anders en heeft daarbij eigen abonnomenten. Een locatie kan slecht 1 abbonoment hebben of meerdere. Een gebruiker kan hierbij zich aansluiten bij een locatie. Hierbij moet de gebruiker een abonnoment kiezen. Een gebruiker kan bij meerdere locaties zijn ingeschrijven. Een gebruiker kan dus meerdere inschrijvingen hebben, waarbij elke inschrijving een bijbehorende locatie en abonnoment heeft.

Naast de functionele requirements zijn er ook een aantal niet functionele requirements of Non-functional requirements. Deze zijn in de onderstaande tabel weergeven.

Datamodel / ERD

Er zijn een aantal models. Met als eerste de User. User heeft altijd een adres. Dit geld ook voor Locatie, ook deze heeft altijd een adres. Een loctatie heeft ook 1 of meerdere abonnomenten, waarbij een abonnoment bij 0 of meerdere locaties behoord. Een user kan zich bij 0 of meerdere locaties inschrijven. Hierbij is ook een abonnoment aan gekoppeld. Verder word daarbij de datum ook opgeslagen
